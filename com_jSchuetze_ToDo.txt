Brainstroming für Joomla-Komponente: com_Zugmitglieder
------------------------------------------------------
Erstellt wird eine Komponente, die die Mitglieder des Zuges verwaltet.
Und zwar in der Form, dass die Daten pro Zugmitglied einmal zentral erfasst
und gespeichert werden und dann für die Generierung der Seiten:
-> Mitgliederliste
-> Zugkönige, Corpskönige
-> Mitgliedermeldung an die Bruderschaft
-> Adressliste des Zuges
verwendet werden.

Informationen, wie Zugsau, Pfänder, etc. werden auch erfasst.

Layout:
Vor den Bruderschaftsbeitritt das St. Sebastianus-Logo.
Vor den TzK.Beitritt das TZK-Logo.

Könnte das Aussehen eines "Personalausweises" haben (mit Schlagschatten).
-> Bild rechts oben
-> TzK Logo links oben
-> Namen und Daten symetrisch verteilt
-> Einzelne Reiter für Mitgliedsdaten, Königschronik, Hubertuschronik


Neue Features:
--------------
Fundus-Verwaltung einbauen
--------------------------
Das Zugeigentum soll erfasst und verwaltet werden.
Zudem muss verwaltet werden, welchem Schützen welche Dinge geliehen wurden.
Leihgaben an einzelne Schützen werden auf einem neuen Reiter bei der Mitgliederliste
ausgegeben. Zudem soll es eine Gesamtübersichtsseite über den Fundus geben.

(beide) Königsseiten:
---------------------
bekommen das Aussehen einer Bildergalerie, die mit tesafilm an die Wand geklebt wurden.
Immer x Bilder nebeneinaner und dann x reihen untereinander.
Boxen bekommen feste Breite und festen Abstand zueinander.
Bild oben mit Tesafilmeffekt, darunter zentriert die Schrift: Periode <br /> Vorame, Titel, Name
evtl. Periode über das Bild schreiben, mal sehen wie das aussieht

-> Einstellbar: wieviel Bilder in einer Zeile angezeigt werden.
-> Einstellbar: ob Bilder links- rechtsbündig oder zentriert dargestellt werden.
-> Einstellbar: ob Chronologisch auf- oder absteigend
-> Einstellbar: Header wie bei den Mitgliedsausweisen
-> Einstellbar: Ob Zug- , Corps- oder Schützenkönigschronik ausgegeben wird
